## Top 12 fixes — ordered by impact ÷ effort

### Tier 1 — Ship NOW (≤ 30 min each, big wins)

1. **🟥 Fix postcss CVE-2026-41305 via `pnpm.overrides`**
   *5 min.* `pnpm.overrides.postcss: "^8.5.12"`. Direct devDep is fine ; transitive via Next/Sentry/Vercel pins 8.4.31 (XSS moderate). Zero-risk.
2. **🟥 Switch `getPublishedKills(500)` → `fn_get_feed_kills(N)` RPC**
   *1h.* The single biggest egress win. Saves ~3 GB/mo on Supabase free tier (currently at ~70-85 % of the 5 GB ceiling). The RPC was defined in migration 001 but never called from `/web/src`. Hot paths : `web/src/app/page.tsx:180`, `web/src/lib/supabase/kills.ts`.
3. **🟥 Migration 051 — security hardening**
   *15 min.* Add `SET search_path = public, pg_temp` to 6 `SECURITY DEFINER` funcs (CVE-class hijack vector). Add `ON DELETE CASCADE` to `game_events.kill_id`, `game_events.moment_id`, `lab_evaluations.kill_id`. Both fix vulnerabilities + prevent future bugs.
4. **🟧 yt-dlp `--concurrent-fragments 8 --throttled-rate 100K`**
   *15 min, zero risk.* 2-3 × VOD download speedup + auto-recovery on YouTube throttle. `worker/services/youtube_dl.py`.
5. **🟨 Bump trivial deps** : `@supabase/ssr` 0.6 → 0.10 (Next 16 cookie bugfixes), `motion` 12.37 → 12.38, `lucide-react` 0.475 → 1.12 (major), drop dead `@types/hls.js`. Bump worker `anthropic` 0.94 → 0.97 + `cryptography` 46 → 47 (CVE).

### Tier 2 — Big wins, larger effort (1-3 h each)

6. **🟧 Lazy-load `three.js` + `matter-js` via `next/dynamic({ ssr:false })`**
   Both deps used in 1-2 files but ship in main bundle (~500 KB chunk). Saves 300-500 KB First Load JS on every route. `web/src/components/sphere/SphereScene.tsx`, `web/src/lib/game/*`.
7. **🟧 Drop `recharts` via `EraComparison.tsx` rewrite**
   ~100 KB on homepage path. Same pattern that already shipped for `PlayerCharts` + `TaggingInsights` (recharts → hand-rolled SVG). Removes recharts dep entirely (the holdout).
8. **🟧 `useOptimistic` migration on community buttons**
   `CommentVote.tsx` reimplements optimistic UI in 80 lines (snapshot+revert). React 19's `useOptimistic` does it in 6. Same for `LikeButton`, `ReportButton`, `DoubleTapHeart`.
9. **🟨 Anthropic Batch API for translator + moderation backfill**
   50 % discount, 24 h SLA. `worker/modules/translator.py` + moderator. Saves ~$2/mo on Haiku spend.

### Tier 3 — Strategic refactors (half-day to day each)

10. **🟥 Migrate `google-generativeai` → `google-genai` SDK**
    `google-generativeai` is OFFICIALLY DEPRECATED, FutureWarning on every import. New `google-genai` SDK supports structured outputs (`response_schema`) which removes our entire JSON-fence-stripping + post-validation regex layer. Effort ~3h. Touch : `worker/services/gemini_client.py`, `worker/modules/analyzer.py`, plus 4 reclip/qc scripts.
11. **🟧 PPR / `cacheComponents` re-enable on 5 public read pages**
    Drop `force-dynamic` from `/`, `/scroll`, `/kill/[id]`, `/player/[slug]`, `/top`. Wrap data fetches in `<Suspense>`. SSG-tier first paint with live data. ~1 day.
12. **🟧 `asyncio.gather()` → `asyncio.TaskGroup`** (Python 3.11+)
    18 occurrences across worker. TaskGroup gives structured concurrency : a single failing R2 upload in `clipper.py` no longer leaves the in-flight ffmpeg encode running uselessly. Major resilience win. ~4h.

## Findings not in the top 12 (lower priority but documented)

### Web
- React 19 asset preload APIs (`preinit`, `preload`, `prefetchDNS`) — 0 usages. Layout uses raw `<link rel>` tags instead.
- 7 raw `<img>` tags should be `next/image` (admin clips library, comment avatars, navbar avatar). Free AVIF + LCP optimization.
- CSS scroll-driven animations (`animation-timeline: scroll()`) — 0 usages. Perfect for KCTimeline horizontal indicator.
- `@starting-style` — native CSS entry animations replace ~90 % of `motion.div` opacity intros.
- `field-sizing: content` for textareas (CommentSheetV2, comment-section).
- Container queries — only `ClipReel` uses them. `KillCard`, `PlayerCard`, `MultiKillBadge` are good candidates.
- `color-mix()` in design tokens replaces ~20 hardcoded rgba variants.
- 13 forms still use `onSubmit` instead of `<form action={serverAction}>`.
- CSP allows `'unsafe-inline'` — Next 16 supports nonce-based CSP via middleware.

### Worker
- Pillow OG generation runs sync — blocks the event loop ~200 ms per image (50/cycle = 10 s of stalled loop). Wrap in `to_thread`.
- `clipper._compute_next_version` does N+1 HTTP per kill before clipping. Batch via `kill_id=in.(...)`.
- `subprocess.run` (sync) in `services/youtube_dl.py` blocks event loop — should use `asyncio.create_subprocess_exec` (already used in `clipper.py` & `ffmpeg_ops.py`, this file is a straggler).
- `structlog.dev.ConsoleRenderer` everywhere → switch to `JSONRenderer` when `KCKILLS_ENV=prod` for Sentry/Datadog parsing.
- No structured Sentry tags on async tasks — `sentry_sdk.set_tag()` per-module would help triage.
- Gemini implicit prompt caching is FREE on `gemini-2.5-pro` + `gemini-3-flash` (NOT `flash-lite`). Switching analyzer to `flash` would cut analyzer cost ~60 %.
- Anthropic prompt caching ($0.10/M cached read vs $1.00/M fresh) on the moderator prompt would save ~70 % of Haiku spend.

### DB
- Drop redundant IVFFlat index on `kills.embedding` (HNSW from migration 018 + IVFFlat from 046 coexist) — at <100k rows the planner picks one, the other costs writes for nothing.
- Convert `fn_update_kill_rating` + `fn_update_comment_count` triggers from O(N) recompute to O(1) incremental UPDATE.
- `fn_record_impression` not rate-limited — a curl loop can pump impression_count infinitely. Add per `(kill_id, IP-hash)` rate-limit OR push to a queue with dedup.
- `getKillsByMatchExternalId` issues 3 parallel SELECTs. Single `fn_get_kills_by_match` RPC saves 2 round-trips.

### Ops
- Backup cron not scheduled (script ready in `worker/scripts/backup_supabase.py`).
- Auto-restart supervisor not registered in Task Scheduler (scaffolding shipped in commit `544b971`).
- pg_cron not enabled — would replace `/api/cron/heartbeat-check` (96 Vercel cron invocations/day → 0).
